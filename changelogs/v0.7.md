## [0.7.1] — 2026-05-08

Infrastructure patch. All architectural achievements belong to v0.7.0.
Test suite: 1,485 passed, 0 failed.

### Added

- **Z204 FORBIDDEN_TERM** (Sprint D100): new credential scanner rule (Exit 2) for forbidden project
  terms declared in machine-local, git-ignored `.zenzic.local.toml`. Two-layer
  architecture: `scan_line_for_forbidden_terms()` in `credentials.py`; `_apply_local_toml()`
  merges patterns additively at load time.
- `zenzic init` and `zenzic init --dev` always create `.zenzic.local.toml` and append
  it to `.gitignore`.
- **Multi-Root Discovery**: VSM no longer bounded by `docs_dir`. Adapters
  declare extra content roots via `get_extra_content_roots(repo_root) -> list[ContentRoot]`.
  `DocusaurusAdapter` auto-detects the `blog/` plugin (static regex + filesystem
  convention fallback). Blog posts are first-class content: broken links inside `blog/`
  and cross-tree links from `docs/` to `blog/` caught by `zenzic check all --strict`.
- **Virtual Routes + `zenzic inspect routes`**: tag pages, tag index,
  paginated indexes, and author profiles are first-class VSM citizens.
  Reverse-Mapping Invariant: `VirtualRoute` with `source_files=frozenset()` raises
  `ValueError`. New finding codes: Z111 `VIRTUAL_ROUTE_BROKEN`, Z113
  `AUTHOR_KEY_COLLISION`, Z114 `LARGE_PAGINATION_SET`. CLI command
  `zenzic inspect routes [--kind physical|virtual|all] [--json]`.
- **Zero-Config Sovereignty**: `absolute_path_allowlist` and
  `[link_validation]` TOML schema removed. `DocusaurusAdapter.get_absolute_url_prefixes()`
  auto-discovers plugin URL prefixes via two pure-Python static-parse passes.
- Zero-Config Asset and Directory defaults: universal toolchain files and build artifact
  directories promoted to `SYSTEM_EXCLUDED_FILE_NAMES` / `SYSTEM_EXCLUDED_DIRS`.

### Changed

- `.zenzic.dev.toml` hard-removed: `_apply_local_toml()` raises `ConfigurationError`
  when `.zenzic.dev.toml` is found at repo root.

### Fixed

- **[ZRT-006] VSM Bypass — Absolute Slug Links**: `DocusaurusAdapter.set_slug_map()` now
  called before `build_vsm()`. Z105 suppression no longer skips VSM lookup for
  project-owned prefixes.

### Removed

- `[link_validation]` TOML schema and `LinkValidationConfig` model — use
  `DocusaurusAdapter.get_absolute_url_prefixes()` instead.
- `.zenzic.dev.toml` support removed entirely.
- `zenzic plugins` command removed; use `zenzic inspect capabilities`.
- Legacy in-engine integration path physically removed; enforcement is delivered
  through standalone CLI execution via Adapters.
- `scripts/map_project.py`.

### Security

- **[ZRT-007] DFA Revolution — Google RE2 Engine**: `CustomRule` patterns now have
  guaranteed O(n) complexity. Backreferences, lookaheads, and lookbehinds rejected at
  load time with `PluginContractError`. `timeout.py` and `signal.SIGALRM` dependency
  deleted — behaviour is now identical on Linux and Windows.
  - `credentials.py` migrated to `re2`: Credential scanner is fully DFA-Pure.
  - Legacy code aliases Z001 and Z009 removed; findings now emit Z101 (`LINK_BROKEN`)
    and Z902 (`ANALYSIS_TIMEOUT`) directly.

## [0.7.0] — 2026-05-07

### Breaking Changes

- `.pre-commit-hooks.yaml`: hook id `zenzic-check-all` removed (no compat alias).
  Replaced by `zenzic-verify` (`--strict --quiet`, 4-Gates posture).
- `just preflight` recipe removed; use `just verify`.
- `validate_same_page_anchors` default changed to `True`.
- `engine` default changed from `"mkdocs"` to `"auto"`.

### Added

- **4-Gates Quality Pipeline**:
  - `just verify`: atomic single entry-point (local pre-push hook + GitHub Actions parity).
  - `just test`: fast TDD loop — `pytest -n auto`, no coverage.
  - `just test-cov`: serial pytest with coverage XML; `fail_under = 80` enforced.
  - Nox declassed to isolated multi-version matrix (3.11/3.12/3.13) only.
  - `pytest-xdist>=3.6` added.
  - `.pre-commit-config.yaml` pre-push stage: `just-verify` local hook.
  - `CONTRIBUTING.md` "4-Gates Standard" section.
- **Z907 I18N_PARITY**:
  - Language-agnostic translation parity scanner: verifies every base-language doc has a
    mirror in each target-language root.
  - `[i18n]` configuration section in `zenzic.toml`: `I18nConfig`, `I18nSource` models;
    `extra_sources` for multi-root repos.
  - `i18n-ignore: true` frontmatter escape hatch.
  - Adaptive parallelism: `ThreadPoolExecutor` when base population > 50 files.
- **Cross-Instance Trust Sovereignty** (later superseded by Zero-Config Sovereignty):
  - `LinkValidationConfig` model with `absolute_path_allowlist` field.
  - `[link_validation]` TOML section.
  - 5 contract tests for allowlist behaviour.
- **CEO-298 — Parallel Fail-Fast**: `concurrent.futures.wait(FIRST_COMPLETED)`. First
  `SecurityFinding` cancels all queued futures.
- **D096 — SARIF Sovereignty + Auto-Discovery**:
  - `discover_engine(repo_root) -> str`: priority Docusaurus → MkDocs → Zensical →
    `"standalone"`.
  - SARIF rules generated dynamically from `codes.py`.
  - `ZenzicExitCode` class: `SUCCESS=0`, `QUALITY=1`, `CREDENTIAL_LEAK=2`, `PATH_TRAVERSAL_FATAL=3`.
  - `--no-external` flag on `check links` and `check all`.
  - `Z906 NO_FILES_FOUND` registered.
- **D095 — Base64 Speculative Decoder**:
  - `_BASE64_CANDIDATE_RE` + `_try_decode_base64()` in `credentials.py`. Seals attack
    vector S2: Base64-encoded credentials in YAML frontmatter trigger Z201 and Exit 2.
  - `os.path.normcase` portability fix in `resolver.py` for macOS/Windows path
    boundary comparisons.
- **D091 — Brand Integrity Rules**:
  - `CircularAnchorRule` (Z107 `CIRCULAR_ANCHOR`): self-referential anchor links.
  - `UntaggedCodeBlockRule` (Z505 `UNTAGGED_CODE_BLOCK`): fenced blocks without
    language specifier. CommonMark closing-fence invariant applied correctly.
  - `BrandObsolescenceRule` (Z905 `BRAND_OBSOLESCENCE`): obsolete release identifiers
    declared in `[project_metadata]` TOML section.
  - `ProjectMetadata` Pydantic model: `release_name`, `obsolete_names`,
    `obsolete_names_exclude_patterns`.
- **Zero-Legacy Policy**: `forbidden_patterns` firewall in `.zenzic.dev.toml`; regression
  guard prevents reintroduction of decoupled brand strings.
- **CLI restructuring**: `cli.py` monolith split into `cli/` package. `zenzic plugins`
  replaced by `zenzic inspect capabilities`.
- **Short flags**: `-c` alias for `--check`, `-f` alias for `--format` (8 commands).
- **`--exclude-url` flag** on `check all` and `check links`: runtime suppression of
  specific external URL prefixes.
- **SARIF output**: all `check` commands support `--format sarif`.
- **Cross-platform CI matrix**: 3×3 (Ubuntu/Windows/macOS × Python 3.11/3.12/3.13).
- **Z107, Z404, Z505, Z905, Z906, Z907** registered in `codes.py`.

### Changed

- `ui.py` → `core/ui.py`; `lab.py` → `cli/_lab.py`; `run_rule()` → `core/rules.py`.
  Core layer never imports from CLI layer.
- Coverage format standardised to JSON (`coverage.json`).
- CI: `.github/workflows/ci.yml` runs `just verify` on Ubuntu/Windows matrix.
- `justfile`: `set shell := ["bash", "-c"]` for consistent Windows runner behaviour.

### Removed

- `scripts/map_project.py`.
- `just preflight` recipe.
- `coverage.json` removed from version control (added to `.gitignore`).

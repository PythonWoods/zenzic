# Changelog — v0.1.x

> Zenzic v0.1.x: initial public release and first iteration.
> For current release history, see the [main Changelog](../CHANGELOG.md).

---

## [0.1.0-alpha.1] — 2026-03-23

### Added

- Native two-pass Markdown link validator (`zenzic check links`): no MkDocs subprocess.
  Pass 1 builds anchor index from ATX headings. Pass 2 validates inline links.
  Pass 3 (`--strict`) validates external URLs via async `httpx` (semaphore 20).
- Quality scorer (`zenzic score`): weighted 0–100 integer across five check categories.
  Supports `--format json`, `--save`, `--fail-under`.
- Regression detector (`zenzic diff`): compares current score against persisted baseline.
- Development server (`zenzic serve`): auto-detects engine, socket-probes for free port,
  runs silent pre-flight before startup.
- Locale-aware orphan detection: auto-excludes `*.{locale}.md` files for `mkdocs-i18n`
  suffix mode.
- `excluded_assets` and `excluded_file_patterns` config fields in `ZenzicConfig`.
- Core library: `zenzic.core.scorer`, `zenzic.core.exceptions`, `zenzic.core.logging`.
- REUSE 3.3 / SPDX compliance across all source files.
- GitHub Actions: `ci.yml`, `release.yml`, `sbom.yml`, `secret-scan.yml`, `security-posture.yml`.
- Pre-commit hooks: ruff, mypy, reuse, zenzic self-check.
- PyPI release via OIDC trusted publishing. Hatch build backend with `hatch-vcs`.
- `zensical` moved to optional extra; lean install by default.

### Changed

- `_detect_engine()` validates binary presence and config file before returning.

### Removed

- `zenzic build` (thin subprocess wrapper; no linting value).
- `detect_generator()` / `DocEngine` replaced by `find_config_file()`.

## [0.1.0] — 2026-03-18

### Added

- `zenzic check links` — strict broken-link and anchor detection via `mkdocs build --strict`.
- `zenzic check orphans` — detect `.md` files missing from `nav`.
- `zenzic check snippets` — syntax-check all fenced Python code blocks.
- `zenzic check placeholders` — flag stub pages and forbidden placeholder text.
- `zenzic check assets` — detect unused images and assets.
- `zenzic check all` — run all checks; supports `--format json`.
- `zenzic.toml` configuration file with Pydantic v2 models.
- `justfile` task runner.
- `examples/broken-docs/` fixture.
- `noxfile.py` developer task runner.
- Full documentation suite (index, architecture, checks reference, configuration).

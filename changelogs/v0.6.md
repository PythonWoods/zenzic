## [0.6.1rc2] — 2026-04-16

### Security

- **[ZRT-006] Unicode format character bypass**: normalizer strips all Unicode category Cf
  characters (ZWJ U+200D, ZWNJ U+200C, ZWSP U+200B) before scanning.
- **[ZRT-006] HTML entity obfuscation**: normalizer decodes `&#NNN;`/`&#xHH;` entities
  via `html.unescape()` before scanning.
- **[ZRT-007] Comment-interleaving bypass**: normalizer strips HTML (`<!-- -->`) and MDX
  (`{/* */}`) comments inserted mid-token.
- **[ZRT-007] Cross-line split-token**: `scan_lines_with_lookback()` maintains a 1-line
  lookback buffer to detect secrets split across consecutive lines (e.g. YAML folded
  scalars). Duplicate suppression via previous-line seen set.

### Added

- `--format json` on individual check commands (`check links`, `check orphans`,
  `check snippets`, `check references`, `check assets`). Uniform `findings`/`summary`
  schema; exit codes preserved.
- GitLab Personal Access Token detection (`glpat-` prefix). Total: 9 credential families.

### Fixed

- JSON exit-code asymmetry in `check orphans` and `check assets`: warnings no longer
  trigger Exit 1 in JSON mode.

## [0.6.1rc1] — 2026-04-15

### Breaking Changes

- `zenzic serve` command removed. Zenzic is 100% subprocess-free.
- Integration model shifted to external Adapter-based analysis via CLI.
  No in-engine runtime hook is required.
- `exclusion_manager` parameter is now mandatory on `walk_files`,
  `iter_markdown_sources`, and all scanner/validator entry points.

### Added

- **Layered Exclusion Manager**: 4-level exclusion hierarchy (System Guardrails >
  Forced Inclusions + VCS > Config > CLI). Pure-Python gitignore parser
  (`VCSIgnoreParser`) with pre-compiled regex patterns.
- New config fields: `respect_vcs_ignore`, `included_dirs`, `included_file_patterns`.
- CLI exclusion flags: `--exclude-dir`, `--include-dir` (repeatable, all check commands).
- Universal Discovery: all file iteration flows through `walk_files` /
  `iter_markdown_sources` from `discovery.py`. Zero `rglob` calls in production modules.
- Adapter Cache: module-level cache keyed by `(engine, docs_root, repo_root)`.
- F4-1 Jailbreak Protection: `_validate_docs_root()` rejects `docs_dir` paths escaping
  the repository root → Exit 3.
- F2-1 Credential Scanner Hardening: lines exceeding 1 MiB silently truncated before regex.
- Adapter architecture formalized for external static analysis over engine
  configuration and content sources.

## [0.6.0a2] — 2026-04-13

### Added

- Glob pattern support for `excluded_assets` via `fnmatch` (`*`, `?`, `[]`, `**`).
- `base_url` in `[build_context]`: explicit site base URL; skips dynamic extraction
  from `docusaurus.config.ts` when set.
- `RouteMetadata` dataclass + `get_route_info()` on `BaseAdapter` protocol. All adapters
  implemented. `build_vsm()` prefers metadata-driven path.
- Centralized frontmatter extraction (`_utils.py`): `extract_frontmatter_slug()`,
  `extract_frontmatter_draft()`, `extract_frontmatter_unlisted()`,
  `extract_frontmatter_tags()`, `build_metadata_cache()`.
- `FileMetadata` dataclass: `slug`, `draft`, `unlisted`, `tags`.
- Credential Scanner IO Middleware: `safe_read_line()` scans frontmatter lines before any parser.
  `CredentialViolation` exception with `SecurityFinding` payload.
- 43 protocol compliance tests.

### Breaking Changes

- `excluded_assets` entries now interpreted as glob patterns (fnmatch). Literal paths
  continue to work (they are valid patterns).

### Fixed

- Docusaurus "dynamic patterns" warning emitted twice when `base_url` was declared.

## [0.6.0a1] — 2026-04-12

### Added

- **Docusaurus v3 Adapter** — first non-MkDocs/Zensical engine adapter.
  Static-AST-like parsing of `docusaurus.config.ts/js`. No Node.js subprocess
  (Pillar 2 compliance). Registered as entry-point `docusaurus`.
  - `baseUrl` extraction: multi-pattern parser (`export default`, `module.exports`,
    `const`/`let`). JS/TS comments stripped before extraction.
  - `routeBasePath` extraction from `@docusaurus/preset-classic` presets and plugins.
  - Frontmatter `slug:` override support (absolute slugs replace full URL; relative
    slugs replace last path segment).
  - Dynamic config detection: async configs, `import()`/`require()` calls → fallback
    to `baseUrl='/'` with user warning.
  - `_`-prefixed file/directory exclusion (`IGNORED`).
  - Docusaurus i18n geography:
    `i18n/{locale}/docusaurus-plugin-content-docs/current/`.
  - Ghost Route detection for locale index pages.
- `from_repo()` factory hook for `DocusaurusAdapter`.
- 65 tests in `test_docusaurus_adapter.py` across 12 test classes.

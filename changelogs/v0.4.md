# Changelog — v0.4.x

> Zenzic v0.4.x: Adapter Pipeline, Virtual Site Map, Ghost Routes, VSM Rule Engine.
> For current release history, see the [main Changelog](../CHANGELOG.md).

## [0.4.0-rc5] — 2026-04-01

### Added

- `examples/zensical-basic/` — canonical example for `engine = "zensical"` using `[project].nav`.
- `_extract_nav_paths()` — pure helper that recursively extracts `.md` paths from a Zensical
  nav list (plain strings, titled pages, nested sections; external URLs silently skipped).
- Nav-aware `classify_route()`: files absent from an explicit `[project].nav` classified
  `ORPHAN_BUT_EXISTING` instead of `REACHABLE`.
- `use_directory_urls` support in `ZensicalAdapter.map_url()`.
- Parallelism documentation in `docs/usage/advanced.md` and `docs/architecture.md`.
- Engine coexistence documentation in `docs/configuration/adapters-config.md` (EN + IT).

### Changed

- `ZensicalAdapter.__init__` pre-computes `_nav_paths`, `_has_explicit_nav`,
  `_use_directory_urls` at construction; `get_nav_paths()` is now an O(1) read.
- Sandbox `tests/sandboxes/zensical/zensical.toml` migrated to `[project]` scope.

### Fixed

- `ZensicalAdapter.get_nav_paths()` was reading legacy `[nav].nav`. Fixed to `[project].nav`.
- `ZensicalAdapter.classify_route()` returned `REACHABLE` for all non-private files
  regardless of nav declaration. Fixed.

## [0.4.0-rc4] — 2026-04-01

### Added

- Ghost Route support (`MkDocsAdapter`): `reconfigure_material: true` marks locale entry
  points `REACHABLE`, eliminating false `ORPHAN_BUT_EXISTING` findings.
- Redundant-config warning when both `reconfigure_material: true` and `extra.alternate`
  are declared simultaneously.
- `Violation` dataclass with `code`, `level`, `context` fields and `as_finding()` converter.
- `BaseRule.check_vsm()` — optional VSM-aware method; default no-op preserves backwards
  compatibility.
- `RuleEngine.run_vsm()` — companion to `run()` for VSM-aware rules.
- `VSMBrokenLinkRule` (Z001) — first VSM-aware built-in rule; validates links against
  `vsm[url].status`.
- `CacheManager` (`zenzic.core.cache`) — content-addressable in-memory cache keyed by
  `SHA256(content + config [+ vsm_snapshot])`. Atomic write via temp-file rename.
- `make_vsm_snapshot_hash()` — deterministic SHA-256 digest of VSM routing state.

### Fixed

- Redundant `extra.alternate` block removed from the project's own `mkdocs.yml`.

## [0.4.0-rc3] — 2026-03-29

> Sprint 7 (i18n Anchor Fix, credential deep-scan) merged with Sprint 8 (Virtual Site Map).

### Added

- **Virtual Site Map (VSM)** (`zenzic.models.vsm`) — `Route` dataclass and `build_vsm()`:
  zero-I/O projection of every `.md` source file to its canonical URL and routing status
  (`REACHABLE`, `ORPHAN_BUT_EXISTING`, `IGNORED`, `CONFLICT`).
- `UNREACHABLE_LINK` detection: links to files that exist on disk but are absent from nav
  are flagged. Disabled automatically for `StandaloneAdapter` and `ZensicalAdapter`.
- Routing collision detection: `_detect_collisions()` marks two source files mapping to
  the same URL as `CONFLICT`.
- `map_url()` and `classify_route()` adapter methods added to `BaseAdapter` protocol,
  `MkDocsAdapter`, `ZensicalAdapter`, and `StandaloneAdapter`.
- Multi-language snippet validation — `check_snippet_content` validates fenced code blocks
  using pure Python parsers: `python/py` → `compile()`; `yaml/yml` → `yaml.safe_load()`;
  `json` → `json.loads()`; `toml` → `tomllib.loads()`. Unsupported tags silently skipped.
- Credential scanner deep-scan for credentials inside fenced blocks: dual-stream architecture
  (`_skip_frontmatter` raw stream for credential scanner; filtered content stream for ref-defs).
- Credential scanner extended to 7 credential families: Stripe live keys (`sk_live_`), Slack tokens
  (`xox[baprs]-`), Google API keys (`AIza`), generic PEM private keys.
- `resolve_anchor()` method on `BaseAdapter` protocol: suppresses `AnchorMissing` when
  the anchor exists in the default-locale equivalent.
- `remap_to_default_locale()` pure utility extracted to `adapters/_utils.py`.
- Visual Snippets for `[[custom_rules]]` findings: offending source line with `│` prefix.
- `strict` and `exit_zero` as first-class `ZenzicConfig` boolean fields.
- JSON output schema stabilised at 7 stable keys.
- Usage docs split: `usage/index.md`, `usage/commands.md`, `usage/advanced.md` (EN + IT).

### Fixed

- `AnchorMissing` branch now calls `adapter.resolve_anchor()`, eliminating false positives
  for translated headings in locale files.
- `MkDocsAdapter.classify_route`: when no `nav:` declared, all files classified `REACHABLE`
  (mirrors MkDocs auto-include). `README.md` remains `IGNORED`.

### Removed

- `_should_suppress_via_i18n_fallback()`, `I18nFallbackConfig`, `_I18N_FALLBACK_DISABLED`
    dead-code marker, `_extract_i18n_fallback_config()` — dead code (~118 lines).

## [0.4.0-rc2] — 2026-03-29

### Added

- `zenzic clean assets` — interactive autofix command; deletes unused assets. Prompts
  `[y/N]`; `-y/--yes` for CI.
- Dynamic Adapter Discovery (`_factory.py`): `get_adapter()` queries
  `importlib.metadata.entry_points(group="zenzic.adapters")` at runtime. Third-party
  adapters registered via Python package entry-points; no Zenzic release required.
- `from_repo()` classmethod pattern: adapters own their own config loading.
- `has_engine_config()` protocol method: replaces `isinstance(adapter, StandaloneAdapter)`;
  scanner has zero dependency on concrete adapter classes.
- `list_adapter_engines() -> list[str]` — sorted list of registered adapter names.
- `--engine ENGINE` flag on `check orphans` and `check all`.
- `[[custom_rules]]` DSL in `zenzic.toml`: per-project lint rules (`id`, `pattern`,
  `message`, `severity`). Patterns compiled once at config-load time.
- `ZensicalAdapter.from_repo()` enforcement: raises `ConfigurationError` when
  `zensical.toml` is absent and `engine = "zensical"` is declared.
- `zenzic init` command with smart engine auto-detection.
- UX Helpful Hint panel when no `zenzic.toml` is present.
- `ZenzicConfig.load()` returns `tuple[ZenzicConfig, bool]`.
- Documentation: `docs/configuration/` split into 4 pages (EN + IT); Writing an Adapter
  guide; MkDocs → Zensical migration guide.

### Changed

- Unknown `engine` string now falls back to `StandaloneAdapter` (was `MkDocsAdapter`).
- `scanner.py` is protocol-only: no concrete adapter imports.
- `output_format` parameter (was `format`) in `check_all`, `score`, `diff`.

### Fixed

- `check all` now runs 7/7 checks: reference pipeline was never invoked.
- `rule_findings` was populated but never rendered in `check references` output.
- YAML frontmatter skipping in `_iter_content_lines()`.
- Image reference false positives: `_RE_REF_LINK` gains `(?<!!)` lookbehind.
- Percent-encoded asset filenames: `unquote()` applied before `normpath`.

## [0.4.0-alpha.1] — 2026-03-26

### Breaking Changes

- i18n Folder Mode migration: docs moved from Suffix Mode (`docs/page.it.md`) to
  Folder Mode (`docs/it/page.md`). URL structure changes.
- `[build_context]` table must be declared last in `zenzic.toml`.

### Added

- `BuildContext` model — new `[build_context]` section in `zenzic.toml`.
- `MkDocsAdapter` — engine adapter with `is_locale_dir()`, `resolve_asset()`,
  `is_shadow_of_nav_page()`.
- `get_adapter()` factory — single entry point for adapter selection.
- Automatic `mkdocs.yml` fallback when `build_context.locales` is empty.
- Nav restructure: five semantic sections, Italian `nav_translations` for all 18 keys.

### Fixed

- 14 `docs/it/**/*.md` false-positive orphans eliminated.
- False-positive broken links on asset paths in locale trees.
- `header.html` SPDX tag: whitespace-stripping Jinja2 tags caused invalid SPDX expression.
- `overrides/` moved to project root so override files are not scanned as docs.

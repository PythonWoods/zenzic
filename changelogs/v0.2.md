# Changelog — v0.2.x

> Zenzic v0.2.x: Two-Pass Reference Pipeline and credential scanner (Z201).
> For current release history, see the [main Changelog](../CHANGELOG.md).

---

## [0.2.1] — 2026-03-24

### Removed

- `zensical.toml` support (Zensical now reads `mkdocs.yml` natively).
- `mkdocs` runtime dependency from `[project.dependencies]`.
- MkDocs plugin entry-point (`mkdocs.plugins`). Use `zenzic check all` in CI.

### Changed

- `find_config_file()` — looks for `mkdocs.yml` only.
- `find_repo_root()` — walks up to `mkdocs.yml` or `.git`; no zensical.toml check.
- `find_orphans()` — TOML-branch removed; always reads `mkdocs.yml`.
- `_detect_engine()` — simplified to single `mkdocs.yml` trigger.
- Build tooling (`noxfile.py`, `justfile`, `deploy-docs.yml`) migrated to `zensical`.

### Added

- Documentation restructure: new `docs/about/` and `docs/reference/` sections (EN + IT).

---

## [0.2.0-alpha.1] — 2026-03-23

### Added

- **Two-Pass Reference Pipeline** (`zenzic check references`).
  - `ReferenceMap` — per-file stateful registry for `[id]: url` definitions.
    CommonMark §4.7 first-wins. Case-insensitive keys. `integrity_score` property.
  - `ReferenceScanner` — three-phase: Harvest → Cross-Check → Integrity Report.
  - `scan_docs_references` / `scan_docs_references_with_links` — high-level orchestrators.
  - Credential scanner (`zenzic.core.shield`) — scans every reference URL during harvesting.
    Three families: OpenAI API key, GitHub token, AWS access key. Exit Code 2 on detection.
    No HTTP requests issued when leaked credentials are present.
  - `LinkValidator` — global URL deduplication across the entire docs tree.
  - Alt-text accessibility check (`check_image_alt_text`) — flags `![](url)` without alt.
- `zenzic.models.references` — canonical module for `ReferenceMap`, `ReferenceFinding`, `IntegrityReport`.

### Changed

- Scanning model: stateless → document-aware (Three-Phase Pipeline with per-file state).
- Memory model: `_iter_content_lines()` generator replaces `.read()` / `.readlines()`.

### Fixed

- Forward Reference Trap — false Dangling Reference errors when `[text][id]` appears before `[id]: url`.
- Reference ID normalisation: leading/trailing whitespace and mixed casing stripped.

### Security

- Exit Code 2 reserved exclusively for credential scan events. Rotate immediately on detection.

# Changelog — v0.3.x

> Zenzic v0.3.x: Build-Aware Intelligence, i18n support, path traversal protection.
> For current release history, see the [main Changelog](../CHANGELOG.md).

---

## [0.3.0-rc3] — 2026-03-25

### Added

- Three reference example projects in `examples/`:
  - `examples/i18n-standard/` — Gold Standard (deep hierarchy, suffix mode, zero absolute links).
  - `examples/broken-docs/` — intentionally broken docs for demo purposes.
  - `examples/security_lab/` — path traversal and credential scan triggers.
- `examples/run_demo.sh` — three-act orchestrator: Standard (pass) → Broken (fail) → Security (block).
- Ghost Artifact demo via `excluded_build_artifacts` in `examples/i18n-standard/`.

### Changed

- ISO 639-1 guard in `_extract_i18n_locale_patterns`: validates locale strings via
  `re.fullmatch(r'[a-z]{2}', locale)`. Version tags, numeric strings, and BCP 47 region
  codes are rejected.

---

## [0.3.0-rc2] — 2026-03-25

### Added

- **Absolute Link Prohibition** — Links starting with `/` trigger a blocking error (Z105).
  Enforces relative paths for hosting-environment portability.
- **Agnostic Suffix-based i18n** — Detects `page.locale.md` suffixes without any
  build-engine plugin dependency.

### Fixed

- i18n Navigation Integrity: migrated project docs from Folder Mode to Suffix Mode.
- Asset Path Symmetry: all relative paths in `.it.md` files are symmetric with `.md` counterparts.

---

## [0.3.0-rc1] — 2026-03-25

### Added

- **Build-Aware Intelligence (i18n)** — Understands MkDocs `i18n` plugin in `folder` mode.
  Links to untranslated pages resolved against default locale before reporting as broken.
- `excluded_build_artifacts` — new `zenzic.toml` field for glob patterns of build-time assets.
- Reference-style link validation (`[text][id]` through full `InMemoryPathResolver` pipeline).
- `I18nFallbackConfig` — internal NamedTuple encoding i18n fallback semantics.
- Tower of Babel test suite (20 i18n scenarios).
- `InMemoryPathResolver` — deterministic, engine-agnostic link resolver. Zero I/O after construction.
- Path traversal protection: `PathTraversal` surfaces as a distinct high-severity outcome.
- `fail_under` field in `zenzic.toml` (0–100) for quality threshold enforcement.
- Dynamic Scoring v2: `zenzic score --save` persists `ScoreReport` JSON snapshot.

### Fixed

- Orphan false positives for files inside locale subdirectories.
- Non-deterministic asset validation: `known_assets: frozenset[str]` pre-map eliminates I/O in hot path.
- Null-safe YAML iteration (`languages: null` handled via `or []` guard).
- Entry Point corrected to `zenzic.main:cli_main`.
- CVE-2026-4539 (Pygments ReDoS): non-reachable in Zenzic's threat model; exemption documented.

### Changed

- CLI: removed all residual MkDocs plugin references. Public API is CLI-only.
- Path traversal protection integrated into `InMemoryPathResolver` core.

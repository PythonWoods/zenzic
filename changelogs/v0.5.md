## [0.5.0a4] — 2026-04-08

### Security

- **[ZRT-001] Frontmatter Bypass (CRITICAL)**: credential scanner was skipping all YAML
  frontmatter. Fix: Dual-Stream architecture — credential scanner stream uses raw `enumerate(fh)`;
  content stream uses `_iter_content_lines()` (frontmatter filtered). Every byte scanned.
- **[ZRT-002] ReDoS + ProcessPoolExecutor Deadlock (HIGH)**: custom rule patterns could
  permanently hang workers. Two defences:
  - `_assert_regex_canary()`: stress-tests each `CustomRule` pattern under a 100ms
    `SIGALRM` watchdog at engine construction. ReDoS patterns raise `PluginContractError`.
  - `future.result(timeout=30)`: timed-out workers produce `Z009 ANALYSIS_TIMEOUT`
    instead of hanging the scan.
- **[ZRT-003] Split-Token Bypass (MEDIUM)**: secrets fragmented across backtick spans
  and concatenation operators evaded detection. Fix: `_normalize_line_for_scan()`
  pre-processor unwraps backtick spans, removes concat operators, collapses table pipes.
  Both raw and normalised forms scanned; `seen` set prevents duplicates.
- **Path Traversal Guard (Exit Code 3)**: links escaping `docs/` to OS system directories
  (`/etc/`, `/root/`, `/var/`, `/proc/`, `/sys/`, `/usr/`) classified
  `PATH_TRAVERSAL_SUSPICIOUS` with `security_incident` severity. Exit 3 takes priority
  over Exit 2 and is never suppressible via `--exit-zero`.
- **Hex-encoded payload pattern**: `(?:\\x[0-9a-fA-F]{2}){3,}` detects hex-encoded payloads.
  Threshold `{3,}` avoids false positives on single hex escapes.

### Changed

- **[ZRT-004] Context-Aware VSM Resolution (MEDIUM)**: `_to_canonical_url()` is now an
  instance method accepting `ResolutionContext(docs_root, source_file)`. Resolves `..`
  segments relative to `source_file.parent`, fixing false negatives for nested files.
- Secure breach reporting pipeline:
  - `_map_credential_to_finding()` — sole authorised bridge from credential scanner to reporter.
  - `_obfuscate_secret()` — partial redaction (first 4 + last 4 chars; full redaction
    for ≤ 8-char secrets). Never passes raw secret to Rich or CI streams.
  - Breach panel rendered before process exits (Exit 2 applied post-render).

### Added

- Graph Integrity: circular link detection via iterative DFS Θ(V+E). `CIRCULAR_LINK`
  finding at `info` severity (advisory, never affects exit codes).
- `INTERNAL_GLOSSARY.toml` — bilingual EN↔IT term registry (15 entries).
- 28 adversarial validation tests in `test_redteam_remediation.py` (ZRT-001 through ZRT-004).

## [0.5.0a3] — 2026-04-04

### Added

- `src/zenzic/ui.py` — `ZenzicPalette` color/emoji constants (`INDIGO`, `SLATE`, `ROSE`,
  `AMBER`), `make_report_header()` banner factory, `emoji()` helper with `--no-color` guard.
- Agnostic Target Support: `zenzic check all <PATH>` accepts a single file or directory.
  `_resolve_target()` and `_apply_target()` helpers; `StandaloneAdapter` auto-selected for
  targets outside `docs_dir`.
- `.pre-commit-hooks.yaml` — first-class pre-commit hook definitions.
- `zenzic.rules` public namespace: `BaseRule`, `RuleFinding`, `CustomRule`, `Violation`,
  `Severity`.
- `run_rule()` test helper for plugin authors.
- Z002 orphan-link warning: `VSMBrokenLinkRule` distinguishes Z001 (broken, error) from
  Z002 (orphan page, warning).
- `zenzic init --plugin <name>`: scaffolds plugin package with `pyproject.toml`,
  `src/<module>/rules.py`, docs fixture, and `zenzic.toml`.
- `zenzic init --pyproject`: embeds config in `[tool.zenzic]`.
- `examples/single-file-target/` and `examples/custom-dir-target/`.
- Anchor torture regression: 1000 cross-linked files.

### Changed

- Native `col_start`/`match_text` propagation ("Breathing"): `_extract_token()` and
  `_TOKEN_RE` removed. Every regex match captures `m.start()` and `m.group()` at
  detection time and propagates end-to-end.
- `LinkInfo` NamedTuple: `(url, lineno, col_start, match_text)`.
- Unified banner counter: `N files (D docs, A assets)`.
- `has_failures` logic: uses filtered findings (`errors > 0`), not raw pre-filter counts.

### Fixed

- mypy: `list[object]` annotations corrected to `list[RenderableType]`.
- Exit-code under target mode: off-target findings no longer trigger false Exit 1.

### Mutation Testing

- Score: **86.7%** (242/279 mutants killed on `rules.py`).
- 80 new mutant-killing tests in `test_rules.py` across 7 test classes.
- Hypothesis property-based testing integrated with `dev`/`ci`/`purity` profiles.

## [0.5.0a1] — 2026-04-02

### Breaking Changes

- `scan_docs_references` now returns `tuple[list[IntegrityReport], list[str]]`.
- `scan_docs_references_parallel` and `scan_docs_references_with_links` removed.
- `RuleEngine` removed; use `AdaptiveRuleEngine` directly.

### Added

- `AdaptiveRuleEngine`: unified rule engine with Hybrid Adaptive Mode (threshold: 50
  files → `ProcessPoolExecutor`).
- `_assert_pickleable(rule)`: validates rules for `ProcessPoolExecutor` serializability
  at construction time; raises `PluginContractError` on failure.
- `ADAPTIVE_PARALLEL_THRESHOLD = 50` module-level constant.
- `PluginContractError` — new exception for rule plugin violations.
- `zenzic.rules` entry-point group in `pyproject.toml`.
- `zenzic plugins list` — lists registered rules with `rule_id`, origin, and class name.
- `pyproject.toml` config support: three-level priority chain (`zenzic.toml` > `[tool.zenzic]`
  > defaults).
- `plugins` config key: explicit allow-list of external rule entry-points.
- `scan_docs_references` `verbose` flag: one-line performance telemetry on stderr.
